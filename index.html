Git Cheat Sheet

Local Repo

# Command Description Example
1 git init ローカルリポジトリを作成する ...
2 git status リポジトリにあるファイルのステータスを確認する ...
3 git add fileName fileNameをステージングステータスに追加 ...
4 git add * ワーキング状態のすべてのファイルをステージング状態に追加 ...
5 git commit -m “Message” “Message”でステージングされた状態をコミットする ...
6 git commit -a -m “Message” “Message”でステージングに入っている、入っていないファイルをコミットする ...
7 git log コミット一覧を確認する ...
8 git reset HEAD filename ステージングにあるfileNameをステージングから外す ...
9 git rm --f filename ステージングにあるfileNameをステージングから外す。且つファイルを削除する ...
10 git rm --cached filename ステージングにあるfileNameをステージングから外す。ファイルを削除しない ...
11 git checkout -- filename filenameの直接のコミットの状態に戻す ...
12 git mv fileName1 fileName2 fileName1名からfileName2名に変更する ...
13 git commit --amend コミット且つ直前のコミットと合体する ...

Remote Repo

# Command Description Example
1 git clone repository [directory] リモートのrepositoryクーロン[directory]を指定すれば[directory]にクーロンする ...
2 git remote –v 現在のリモート情報を確認する ...
3 git remote show name 現在のリモートのnameを確認する ...
4 git remote add name repository 現在のローカルをnameでリモートrepository追加する ...
5 git remote rename name1 name2 リモート名をname1からname2に変更する ...
6 git remote remove name nameリモートを削除する ...
7 git pull name branch リモートリポジトリとロカールバージョンが違えばマージする ...
8 git push name branch ロカールブランチをリモートリポジトリにプッシュする ...

Branch

# Command Description Example
BRANCH
1 git branch branchname branchnameブランチを作成する ...
2 git log --oneline --decorate ログを確認 ...
3 git checkout branchname ワークスペースをbranchnameに変換する ...
4 git checkout -b branchname branchnameを新規作成、且つbranchnameにワークスペースを変換する ...
5 git branch ブランチ一覧を確認する, * 付いているブランチは現在作業しているブランチ ...
6 git branch -v ブランチそれぞれの最新コミットを確認する ...
7 git merge branchname 現在のブランチを branchnameにマージする ...
8 git branch --merged マージされているブランチ一覧を確認 ...
9 git branch --no-merged マージされていないブランチ一覧を確認 ...
10 git branch -d branchname マージされたbranchnameブランチを削除する ...
11 git branch -D branchname マージされていないbranchnameブランチを削除する ...
12 git branch -m oldname newname oldenameからnewnameに名前を変更する ...
13 git branch -m newname 現在のブランチ名をnewnameに変更する ...
TAG
1 git tag tagname 新規tagnameブランチを作成、コメントなし ...
2 git tag -a tagname -m “Mesage” 新規tagnameブランチを作成、コメントあり ...
3 git tag -a tagname commitID -m “Mesage” 新規tagnameを指定したcommitIDに作成する ...
4 git tag タグの一覧を確認 ...
5 git tag -n タグの一覧且つメッセージの内容を確認 ...
6 git show tagname tagnameブランチの情報を確認 ...
7 git tag –d tagname tagnameブランチを削除 ...
8 git checkout –b branchname tagname 新規branchnameブランチを存在しているtagnameから作成する ...
9 git push name tagname tagnameタグをnameリモートに追加する ...
10 git push name --tags ロカールにあるタグを全部nameリモートに追加する ...
11 git push name -d tagname nameリポジトリにあるtagnameブランチを削除する ...
REMOTE BRANCH
1 git remote show remote_name remote_nameリポジトリにあるブランチを確認 ...
2 git ls-remote remote_name remote_nameにあるブランチ且つタグを確認 ...
3 git clone -b branch remote_url Clone một Branch chỉ định từ Remote repo ...
4 git checkout remote_name/branch_name Chuyển đổi làm việc sang nhánh remote ...
5 git push origin :branch_name Xóa một Branch từ Remote repo ...
REBASE
1 git rebase branch_name Chạy lại lịch sử commit của branch_name trên nhánh hiện tại ...

COMPARISONS
1 git diff
git difftool		 Staging vs Working (L-R) ...
2 git diff HEAD Local Repo vs Working (L-R) ...
3 git diff --staged HEAD
git diff –cached		 Local Repo vs Staging Area (L-R) ...
4 git diff master origin/master Local Repo vs Remote Repo (L-R) ...

6 git diff tagname vs tagname Tag vs Tag ...
STASH

3 git stash pop [stash@{1}]
git stash apply stash@{1}		 Áp dụng thay đổi từ stash ...
4 git stash drop stash@{1}
git stash clear		 Xóa các thay đổi không cần thiết ...
